 🎯 R2構造をサイトに反映する手法（3案）

  案1: ビルド時にR2を自動スキャン ⭐️ 最推奨

  仕組み:
  npm run build
    ↓
  1. R2のフォルダ構造を全スキャン
  2. カテゴリフォルダ → 記事フォルダ → 画像枚数を自動検出
  3. posts.ts を自動生成
  4. Next.jsビルド

  必要なもの:
  - R2 API キー（読み取り専用でOK）
  - generate-posts.js の改修（30-60分）

  メリット:
  - ✅ R2が唯一の真実 - フォルダを作るだけで自動反映
  - ✅ JSONファイル不要
  - ✅ 画像枚数も自動カウント
  - ✅ 記事名・カテゴリ名がフォルダ名から自動取得
  - ✅ 人的ミスゼロ

  デメリット:
  - ❌ ビルド時間が少し増える（R2 API呼び出し）
  - ❌ 記事タイトル・説明文はフォルダ名から推測（調整必要なら別途設定）
  - ❌ R2 APIキーが必要

  コスト:
  - 金銭: 無料（R2 API: 100万リクエスト/月まで無料）
  - 工数: 1-2時間（スクリプト改修）
